Automatic Doctor Help

Hello, and thank you for your purchase of the Med-Tek Pip-Boy Automatic Doctor. Because your life is too important to leave in your own hands.

This guide will help you understand the Auto Doc Pip-Boy application and how it can benefit you and your life on the front lines.

Interface

Health [ ##/## ] [ ##/## ] << Status >>

The first set of brackets will show the current health value, and the maximum health value.

The second set of brackets will show the current health percentage, and the Health % Use value from the Stimpak settings.

After the bracket values, up to two messages can be shown:

<< Irradiated >>
Means that radiation damage has been done, but radiation is no longer being detected and causing further damage.
<< Radiation >>
Means that radiation is being detected and is causing damage.
<< Addicted >>
Means that a dependency on drugs or alcohol has been detected.

Chem Stock [ AA(##/##) ]

This will show (upon activating) how many of what substances are available, and what the current stock limitation is set to for that substance.

The following values can be seen inside of the brackets:

Disabled
All substance use has been disabled.
ST
Stimpak use is enabled.
MX
Med-X use is enabled.
RA
RadAway use is enabled.
RX
Rad-X use is enabled.
AD
Addictol use is enabled.

Chem Applied [ Chem ]

This will show a list of all substances detected in use.

General Settings

Use Delay Timer

Places a delay on using a drug again. User vitals might not update immediately after a drug has been taken, so an imbalance might still exist. This will cause the Auto Doc to use more drugs. The delay will prevent over use.

Stimpak Settings

Health % Use

When the health drops below this percentage, a Stimpak will be taken.

Stock Limitation

The Auto Doc will be allowed to use a Stimpak, only if the number available in the users inventory exceeds this value.

Med-X Settings

Use Type

Med-X will be applied:

When Damaged
When the health value is less than the last value check.
On Health % Use
When the health percentage is less than the Stimpak Health % Use value.
At 3/4 Health
When the health value is less than 3/4 full health.
At 1/2 Health
When the health value is less than 1/2 full health.
At 1/4 Health
When the health value is less than 1/4 full health.

Stock Limitation

The Auto Doc will be allowed to use Med-X, only if the number available in the users inventory exceeds this value.

RadAway Settings

Use Type

RadAway will be applied:

When Irradiated
When radiation has been detected.
When Stimpak Heals Nothing
When radiation has damaged health sufficiently and Stimpak use is enabled, a Stimpak will be used. If at any point during the regeneration process health stops recovering, but is below the maximum value, the remaining damage is likely due to radiation damage. A RadAway will be used immediately to reverse the damage and allow the Stimpak to continue its regeneration.

Stock Limitation

The Auto Doc will be allowed to use RadAway, only if the number available in the users inventory exceeds this value.

Rad-X Settings

Use Type

Rad-X will be applied:

When Irradiated
When radiation has been detected.
On RadAway Use
Whenever RadAway is used, Rad-X will be used as well.

Stock Limitation

The Auto Doc will be allowed to use Rad-X, only if the number available in the users inventory exceeds this value.

Addictol

Stock Limitation

The Auto Doc will be allowed to use Addictol, only if the number available in the users inventory exceeds this value.

Widget Information

Radiation Detection

Radiation detection is… weird. The information being sent to the companion application does not include a value of how much a player has sustained in radiation damage. All it simply shows is current health being less than max health, as if the player was damaged normally. However, what can be detected is any effect that applies radiation damage to the player. So when you enter a radiation field, or during a rad storm, or even when damaged by enemies, an effect (likely a Spell Effect) is applied for a duration and begins to damage the player. So what this means for the Auto Doc right now, is that when the widget and corresponding drugs have been activated, it will not be able to detect existing radiation damage. What it will be able to do is detect when radiation has been applied, and when radiation had been active but not treated. Upon using a RadAway, the flag for being irradiated will be removed. Unfortunately if the RadAway did not entirely remove all radiation damage, no further RadAway will be applied until radiation has been detected.

Aquaperson, Radiation Damage, and the Language Barrier

In order to prevent the use of radiation drugs when the player enters water with the Aquaperson perk, the widget needs to make a couple of checks on currently applied status effects. The Aquaperson perk applies an effect called Waterbreathing to the player. Next, when the player is in deep enough water an effect called Water Radiation is applied. The Auto Doc uses these two effects to determine if it should, or should not apply radiation drugs. Unlike things other things such as items, effects do not have form values associated with them. This means that the widget needs to rely on English words to determine if an effect is active. While the Aquaperson effect applies a sub effect called waterbreathing (there is a chance that this effect is not different in alternate languages because it is not visibly seen), the water radiation effect uses a generic sub effect name of rads. The Auto Doc already checks for any effects with the rads sub effect to detect radiation normally. So without the ability to check for a difference, with or without the Aquaperson perk, radiation will be detected and the appropriate drugs will be taken. If an alternate language version of Fallout 4 can check on this, it would be appreciated.
